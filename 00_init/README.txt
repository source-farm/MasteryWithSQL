--------------------------------------------------------------------------------
 Первоначальная настройка
--------------------------------------------------------------------------------

После установки PostgreSQL в системе создаётся пользователь postgres, из-под
которого и можно подсоединяться к СУБД. Сделать это можно, например, так

sudo -u postgres psql

Эта команда выполняет команду "psql"из-под пользователя postgres. По-умолчанию
PostgreSQL использует понятие "роль" (role) для аутентификации и авторизации.
После установки PostgreSQL использует схему аутентификации ident, которая
означает что название каждой роли внутри СУБД связывается с пользователем с
таким же названием в системе и этот пользователь может делать то, что разрешено
этой ролью. Роль postgres и пользователь postgres создаются при установке
PostgreSQL (также автоматически создаётся база данных postgres) и этот
пользователь может делать всё что угодно в СУБД.

Чтобы поменять пароль пользователя postgres внутри СУБД (не пароль учётной
записи пользователя postgres в системе) нужно набрать приведённую выше команду,
а затем следующую

# \password postgres

и ввести новый пароль. Через этот пароль можно соединяться с СУБД из таких
инструментов как DBeaver и т.д.

--------------------------------------------------------------------------------
 Создание обучающей базы
--------------------------------------------------------------------------------

Скрипт videoezy.sql содержит код для заполнение БД данными. Можно создать БД
videoezy (create database videoezy) и применить этот скрипт к ней.
